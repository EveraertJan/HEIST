RENTAL SYSTEM
-------------
✅ Database: rentals table (status: requested, approved, finalized, rejected)
✅ Backend:
   - RentalRepository: CRUD, availability check, date conflict detection
   - RentalService: Create requests, approve, reject, finalize rentals
   - Routes: POST /rentals, GET /rentals/my-rentals, GET /rentals/pending (admin)
   - PUT /rentals/:uuid/approve, /reject, /finalize (admin only)
   - POST /rentals/check-availability (public)
✅ Frontend:
   - RentArtwork: Request rental form with address, phone, dates
   - MyRentals: View user's rental history
   - AdminRentals: Approve/reject requests, finalize returns
   - Navbar: "My Rentals" and "Manage Rentals" (admin) links
   - ArtworkDetail: "Rent This Artwork" button
✅ Features:
   - Users cannot rent if artwork already requested/approved for same dates
   - Real-time availability checking
   - Admin approval workflow
   - Rental status tracking (requested → approved → finalized)
